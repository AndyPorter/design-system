Work in progress

Work in flight which we'd like to agree and publish soon, such as new advice, that could be improved with more people involved.

Footer for staff systems

A drafted footer specification for staff systems (only). A footer is a common element we all need and we're being led by needing to document advice for footer links (new window or not?)

Current status: Andy and Laura are creating the first draft. Once we're happy, it will be sent to the practice for review. Note: this creation/publishing process is also being drafted/worked out.

Help by:

  • suggesting the collective name we should use for software created for staff - staff systems? internal systems? casework systems? (casework would limit it to caseworkers)
  • adding more examples of existing footers: how they work, evidence about their effectiveness, things you'd like to test.

Links

Drafting advice for using links. A general document about the different types of links. This came about because the question about opening footer links in a new window or not came up on slack again, and it wasn't documented last time.

It made sense to combine this with the footer advice as thats where the links live.

Current status: Andy and Laura are creating the first draft that works with the footer advice. Once we're happy, both will be sent to the practice for review.

Help by:

  • we don't have a collective document on links but we do have an action links document - add and update: how they work, evidence about their effectiveness, things you'd like to test.
  • start documenting anything different (to GOV advice) about links in your system or service (message Andy P to ask where to store this, or for a template)

Process for creating and publishing advice

How do we (the practice) agree the design advice that should be followed?

As we started drafting the footer we discussed how do we get practice agreement? But we dont want deciding this process to hold back publishing advice (as everything can be iterated, and you best learn by doing). The process will be public (on this site) and commentable, like every document (Work in the open).

Current status: Andy and Laura are documenting the first version of this process. We're following it, and once we're happy, will be shared with the practice for review/feedback.

Help by:

  • considering how you would like to be involved in determining advice that gets created and published. What do you need to know? what capacity do you have for drafting, reviewing, testing, etc. What can you contribute?
  • commenting on version 1 of the process
